# 產品需求文件 (PRD) - 食譜收藏系統

## 1. 專案概述

- **背景與動機**：希望能有一個專屬的空間來記錄自己喜歡或是開發的食譜，隨時尋找並記錄每次烹飪的配方與詳細步驟，方便日後查閱與製作。
- **目標用戶**：我自己（個人使用的食譜管理系統）。
- **核心價值主張**：提供一個簡單、直覺的介面，快速紀錄食譜的材料與步驟，並能輕鬆透過搜尋找到想要的食譜。

## 2. 功能需求

以下列出系統的主要功能，並以使用者故事（User Story）的格式描述：

1. **新增食譜 (Add Recipe)**
   - 作為 **系統使用者**，我希望 **能新增一份食譜，包含名稱、材料清單與詳細步驟**，以便 **我能長期保存這些配方**。
2. **檢視食譜 (View Recipe)**
   - 作為 **系統使用者**，我希望 **能點擊某一食譜並完整檢視其材料與步驟**，以便 **在烹飪時能直接對照著做**。
3. **尋找食譜 (Search Recipes)**
   - 作為 **系統使用者**，我希望 **能透過關鍵字搜尋食譜名稱或食材**，以便 **快速找到我今天想做的菜**。
4. **編輯食譜 (Edit Recipe)**
   - 作為 **系統使用者**，我希望 **能修改既有食譜的內容**，以便 **在我改良配方或發現步驟錯誤時進行更新**。
5. **刪除食譜 (Delete Recipe)**
   - 作為 **系統使用者**，我希望 **能刪除已經不再需要的食譜**，以便 **保持系統與列表的整潔**。

## 3. 非功能需求

- **技術限制**：系統將依照標準建議使用 Flask（後端） + Jinja2（前端模板） + SQLite（資料庫）進行開發。
- **效能與安全考量**：
  - 系統主要為本機端或個人使用，注重使用的流暢度。
  - 對於文字輸入（如食譜名稱、材料、步驟）需進行基礎防護（如防範 SQL Injection 與 XSS），確保資料存取安全。
  - 網頁介面需支援響應式設計（RWD），確保在手機上看食譜做菜時也能輕鬆閱讀。

## 4. MVP 範圍 (Minimum Viable Product)

| 優先級別 | 功能項目 |
| :--- | :--- |
| **Must Have<br>(必須有)** | 1. 建立包含名稱、材料、步驟的食譜<br>2. 食譜列表總覽<br>3. 檢視個別食譜詳細內容<br>4. 簡易關鍵字搜尋（尋找食譜） |
| **Should Have<br>(應該有)** | 1. 編輯已建立的食譜<br>2. 刪除食譜 |
| **Nice to Have<br>(可以有)** | 1. 食譜分類或標籤功能<br>2. 為食譜上傳圖片<br>3. 食譜匯出與分享功能 |
